    class Person

    People

    Fields

    1. name – string, unique, mandatory

    2. email – mandatory, validated

    3. description – text, optional

    4. about_me – text, user supplied

    5. twitter_handler – twitter screen name

    6. settings – text, internal, used for user preferences

    7. person_type – string. One of 'anonymous', 'registered', 'subscriber', 'admin', 'talkinvite'.

    Not 'type', which will cause rails to spin on its head three times widdershins & then croak

    Possible additional fields

    1. twitter_id – this is stable, unlike the twitter screen name which can change

    2. current_location_id

    3. home_location_id

    4. other fields as in twitter api
